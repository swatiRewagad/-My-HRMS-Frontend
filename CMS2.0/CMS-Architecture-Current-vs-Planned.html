  • Keycloak Authentication -- OAuth2/JWT with RBAC (citizen, officer, admin roles) and SSO integration
  • Outbox Publisher Service -- Dedicated service polling Oracle outbox table and publishing events to Kafka topics
  • Assignment Service -- Standalone microservice with Drools KieContainer for complex rule-based complaint routing
  • Apache Kafka -- Event backbone connecting all microservices with topic-based pub/sub
  • OpenSearch -- Full-text search with auto-indexing via Kafka consumer
  • Notification Service -- Email (SMTP) and SMS (gateway) delivery with template engine

4 Architecture Decision Records (ADRs)

▼
ADR-1: H2 Instead of Oracle for Dev-Local ▼
Context: Oracle XE requires installation, licensing, and 2GB+ RAM. Not practical for rapid local development.
Decision: Use H2 with Oracle compatibility mode for dev-local profile.
Consequence: Data lost on restart. Some Oracle-specific SQL may not work. Schema auto-created from JPA entities.
Production: Will use Oracle XE 21 with Flyway migrations.
ADR-2: Direct Kafka Instead of Outbox Pattern ▼
Context: Outbox pattern prevents message loss if app crashes between DB commit and Kafka send. Critical for production reliability.
Decision: Skip outbox in dev-local, publish directly to Kafka after DB save via DirectKafkaEventPublisher.
Consequence: Theoretical message loss on crash (acceptable in dev). Simpler code path for debugging.
Production: Will use OutboxEventPublisher + cms-outbox-publisher service with ShedLock polling.
ADR-3: In-Memory Workflow Instead of jBPM ▼
Context: jBPM needs Oracle tables (PROCESSINSTANCEINFO, SESSIONINFO, etc.) and full persistence unit configuration.
Decision: DevLocalWorkflowService with ConcurrentHashMap simulates state transitions and assignment.
Consequence: No persistent process instances, no timer events, no visual BPMN execution. State lost on restart.
Production: Will use jBPM RuntimeManager with Oracle persistence and full BPMN2 process definitions.
ADR-4: POST Instead of PATCH for Inter-Service Calls ▼
Context: Java's HttpURLConnection (used by RestTemplate) doesn't support PATCH on Windows. Calls fail with ProtocolException.
Decision: Added duplicate POST endpoints alongside PATCH for internal service-to-service communication.
Alternative: HttpComponentsClientHttpRequestFactory (adds Apache HttpClient dependency). Not adopted to minimize dependencies.
Consequence: Both POST and PATCH work. POST used by internal services, PATCH available for standard REST clients.
ADR-5: Embedded Assignment Logic ▼
Context: Assignment Service requires Drools KieContainer + Kafka consumer setup. Additional microservice to maintain in dev.
Decision: DevLocalWorkflowService handles assignment via simple switch/case on complaint category.
Consequence: No complex rule-based assignment in dev, but end-to-end flow works. One less service to start.
Production: Enable standalone Assignment Service (8085) with full Drools rules for multi-criteria routing.
