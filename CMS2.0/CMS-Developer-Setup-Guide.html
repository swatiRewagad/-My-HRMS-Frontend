5. Local Infrastructure Setup

Option A: Docker Compose (Recommended - All-in-One)

cd CMS2.0/deployment/docker
docker-compose up -d

# This starts: Oracle, Kafka, Keycloak, OpenSearch, Prometheus, Grafana, Kafka UI
# Wait 2-3 minutes for all services to be healthy
docker-compose ps   # Verify all are "Up (healthy)"

Option B: Manual (Without Docker)

5.1 Keycloak

# Download: https://github.com/keycloak/keycloak/releases/tag/26.0.0
# Extract to C:\tools\keycloak-26.0.0

cd C:\tools\keycloak-26.0.0
bin\kc.bat start-dev --http-port=8180

6. Backend Setup (IntelliJ IDEA)

IntelliJ Configuration

Install Required Plugins

  • Lombok - File > Settings > Plugins > search "Lombok" > Install
  • Spring Boot (bundled in Ultimate)
  • MapStruct Support (optional but helpful)

Enable Annotation Processing

File > Settings > Build, Execution, Deployment > Compiler > Annotation Processors > Enable annotation processing ✓

Set JDK 21

File > Project Structure > Project > SDK: temurin-21 (or Oracle JDK 21)

File > Project Structure > Project > Language Level: 21

Import Project

File > Open > select CMS2.0/ folder > Import as Maven project

Wait for Maven to resolve all dependencies (may take 5-10 minutes first time)

Configure Run Configuration for cms-backend

  • Run > Edit Configurations > + > Spring Boot
  • Main class: com.hrms.cms.CmsApplication
  • Active profiles: dev
  • VM options: -Dspring.profiles.active=dev
  • Working directory: CMS2.0/cms-backend

Running cms-backend (Command Line)

cd CMS2.0/cms-backend

# Build
mvn clean install -DskipTests

# Run with dev profile (H2 in-memory, no MySQL needed)
mvn spring-boot:run -Dspring-boot.run.profiles=dev

# OR with MySQL (default profile)
mvn spring-boot:run

# Verify: http://localhost:8082/actuator/health
Dev Profile Benefits: Uses H2 in-memory DB (no MySQL install needed), auto-creates tables, embedded Hazelcast cache (no external cache needed), includes test data via DataInitializer.

Running Microservices (Full Stack)

# Build all modules from parent POM
cd CMS2.0
mvn clean install -DskipTests

# Run each service with dev-local profile (each in separate terminal)
cd cms-api-gateway && mvn spring-boot:run -Dspring-boot.run.profiles=dev-local
cd cms-ingestion-service && mvn spring-boot:run -Dspring-boot.run.profiles=dev-local
cd cms-workflow-service && mvn spring-boot:run -Dspring-boot.run.profiles=dev-local
cd cms-assignment-service && mvn spring-boot:run -Dspring-boot.run.profiles=dev-local
cd cms-search-service && mvn spring-boot:run -Dspring-boot.run.profiles=dev-local
cd cms-notification-service && mvn spring-boot:run -Dspring-boot.run.profiles=dev-local

# Note: dev-local profile uses H2 + localhost Kafka + localhost OpenSearch
Minimum for local dev: You only NEED cms-backend (dev profile) + frontend. Other microservices are for integration testing.

12. OpenShift Deployment

Prerequisites for OpenShift

  • oc CLI installed and logged in to cluster
  • Access to internal container registry: registry.rbi.internal/cms/
  • Namespace rbi-cms created
  • Oracle DB provisioned on cluster
  • Kafka cluster (AMQ Streams / Strimzi) available
  • Keycloak operator deployed

Step-by-Step Deployment

Create Namespace & Configs

oc apply -f deployment/openshift/namespace.yaml
oc apply -f deployment/openshift/configmap-common.yaml
oc apply -f deployment/openshift/secret-db.yaml

Build & Push Docker Images

# For each service:
cd CMS2.0/cms-api-gateway
mvn clean package -DskipTests
docker build -t registry.rbi.internal/cms/api-gateway:1.0.0-SNAPSHOT .
docker push registry.rbi.internal/cms/api-gateway:1.0.0-SNAPSHOT

# Repeat for all services...

Deploy Services

oc apply -f deployment/openshift/deployment-api-gateway.yaml
oc apply -f deployment/openshift/deployment-ingestion-service.yaml
oc apply -f deployment/openshift/deployment-workflow-service.yaml
oc apply -f deployment/openshift/deployment-outbox-publisher.yaml
oc apply -f deployment/openshift/deployment-eligibility-service.yaml

Deploy Frontend (Nginx)

cd CMS2.0/cms-portal-frontend
ng build --configuration production

# Dockerfile for frontend:
FROM nginx:alpine
COPY dist/cms-portal-frontend/browser/ /usr/share/nginx/html/
COPY nginx.conf /etc/nginx/conf.d/default.conf

docker build -t registry.rbi.internal/cms/portal-frontend:1.0.0 .
docker push registry.rbi.internal/cms/portal-frontend:1.0.0

Verify Deployment

oc get pods -n rbi-cms
oc get routes -n rbi-cms

14. Troubleshooting

Common Issues

Port already in use (8082)
netstat -ano | findstr :8082 then taskkill /PID <pid> /F
MySQL not running / Connection refused
Use dev profile: mvn spring-boot:run -Dspring-boot.run.profiles=dev (uses H2 in-memory DB)
Lombok not working in IntelliJ
1. Install Lombok plugin
2. Enable annotation processing (Settings > Build > Compiler > Annotation Processors)
3. Rebuild project (Build > Rebuild Project)
Angular CLI version mismatch
npm install -g @angular/cli@21 (must match project version)
Keycloak "invalid redirect_uri"
In Keycloak admin > Clients > cms-portal > Valid Redirect URIs: add http://localhost:4300/*
Kafka connection refused
Services with missing-topics-fatal: false will start without Kafka. For full flow, ensure Kafka is running on 9092.
CORS errors on API calls
Backend has CORS configured for localhost:4300. If using different port, update backend CORS config.
Maven build fails on cms-common
Build parent first: cd CMS2.0 && mvn clean install -DskipTests (installs all modules to local .m2)
