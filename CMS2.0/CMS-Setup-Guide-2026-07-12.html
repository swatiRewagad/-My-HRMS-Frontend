3. Option A — Local Machine (Without Docker)

Setup Time: ~30 minutes

This runs only the cms-backend + cms-portal-frontend with Oracle XE. Kafka, OpenSearch, and OCR features will degrade gracefully (logged warnings, no crashes).

Step 1: Install Oracle XE 21c

# Windows — Download Oracle XE 21c from:
# https://www.oracle.com/database/technologies/xe-downloads.html

# Run the installer (OracleXE213_Win64.zip)
# Set SYS/SYSTEM password during install (e.g., oracle123)
# Default port: 1521, PDB: XEPDB1

# After installation, Oracle service starts automatically.
# Verify it's running:
lsnrctl status

Step 1b: Create Application User

# Connect as SYSTEM to the pluggable database:
sqlplus system/oracle123@localhost:1521/XEPDB1

-- Create CMS application user:
CREATE USER cms_app IDENTIFIED BY cms_app_password
  DEFAULT TABLESPACE USERS
  QUOTA UNLIMITED ON USERS;

GRANT CONNECT, RESOURCE TO cms_app;
GRANT CREATE VIEW, CREATE SEQUENCE, CREATE TRIGGER TO cms_app;

EXIT;
If you already have Oracle XE installed at C:\tools\ or as a Windows service, skip the install. Just create the cms_app user in XEPDB1.

Step 2: Install & Start Keycloak

# Download Keycloak 26.0.0
# Extract to C:\tools\keycloak-26.0.0

# Start in dev mode:
cd C:\tools\keycloak-26.0.0
bin\kc.bat start-dev --http-port=8180

# Admin console: http://localhost:8180
# Default: admin / admin

Step 3: Start CMS Backend

cd CMS2.0/cms-backend

# Set environment variables (Windows CMD):
set DB_URL=jdbc:oracle:thin:@localhost:1521/XEPDB1
set DB_USERNAME=cms_app
set DB_PASSWORD=cms_app_password
set SPRING_JPA_DATABASE_PLATFORM=org.hibernate.dialect.OracleDialect
set SPRING_DATASOURCE_DRIVER_CLASS_NAME=oracle.jdbc.OracleDriver
set KEYCLOAK_URL=http://localhost:8180
set KEYCLOAK_ADMIN_USER=admin
set KEYCLOAK_ADMIN_PASSWORD=admin
set KAFKA_MAX_BLOCK_MS=3000

# Run:
mvn spring-boot:run

PowerShell alternative:

$env:DB_URL="jdbc:oracle:thin:@localhost:1521/XEPDB1"
$env:DB_USERNAME="cms_app"
$env:DB_PASSWORD="cms_app_password"
$env:SPRING_JPA_DATABASE_PLATFORM="org.hibernate.dialect.OracleDialect"
$env:SPRING_DATASOURCE_DRIVER_CLASS_NAME="oracle.jdbc.OracleDriver"
$env:KEYCLOAK_URL="http://localhost:8180"
$env:KEYCLOAK_ADMIN_USER="admin"
$env:KEYCLOAK_ADMIN_PASSWORD="admin"
$env:KAFKA_MAX_BLOCK_MS="3000"

mvn spring-boot:run
Kafka is optional for local dev. The backend will log warnings about Kafka connectivity but won't crash. Set KAFKA_MAX_BLOCK_MS=3000 to fail fast (3s timeout instead of 60s default).
Ensure the Oracle JDBC driver is in pom.xml. It should already be present:
<dependency><groupId>com.oracle.database.jdbc</groupId><artifactId>ojdbc11</artifactId><scope>runtime</scope></dependency>

Step 4: Start CMS Portal Frontend

cd CMS2.0/cms-portal-frontend

# Install dependencies:
npm install

# Start dev server (proxies /api to backend on 8082):
npm start
# OR: ng serve --port 4300
Frontend: http://localhost:4300 | Backend API: http://localhost:8082 | Keycloak: http://localhost:8180

Step 5 (Optional): Start OpenSearch Locally

# Download OpenSearch 2.18.0
# Extract to C:\tools\opensearch-2.18.0

# Disable security for local dev:
# In config/opensearch.yml add:
#   plugins.security.disabled: true

cd C:\tools\opensearch-2.18.0
bin\opensearch.bat

Step 6 (Optional): Start Kafka Locally

# Download Apache Kafka 3.7.0
# Extract to C:\tools\kafka

# Start KRaft mode (no Zookeeper):
cd C:\tools\kafka

# Generate cluster ID (first time only):
bin\windows\kafka-storage.bat random-uuid
# Output: some-uuid-here

# Format storage (first time only):
bin\windows\kafka-storage.bat format -t some-uuid-here -c config\kraft\server.properties

# Start:
bin\windows\kafka-server-start.bat config\kraft\server.properties

4. Option B — Docker Desktop (docker-compose)

Setup Time: ~15 minutes (first pull ~10 min)

This runs the full infrastructure stack (Oracle, Kafka, Keycloak, OpenSearch, Prometheus, Grafana) in containers. You still run the backend and frontend on host for hot-reload during development.

Step 1: Start Infrastructure Containers

cd CMS2.0/deployment/docker

# Pull and start all services:
docker-compose up -d

# Watch logs (wait for all services to become healthy):
docker-compose logs -f

# Check health status:
docker-compose ps

# Check all containers are "healthy":
docker-compose ps

docker exec -it cms-oracle bash -c "sqlplus cms_app/cms_app_password@//localhost:1521/XEPDB1 <<< 'SELECT 1 FROM DUAL;'"

# Test OpenSearch:
curl http://localhost:9200/_cluster/health

# Test Keycloak:
curl http://localhost:8180/realms/master

Step 3: Start CMS Backend (on host, connects to Docker infra)

cd CMS2.0/cms-backend

# Environment points to Docker containers (Oracle on port 1521):
set DB_URL=jdbc:oracle:thin:@localhost:1521/XEPDB1
set DB_USERNAME=cms_app

mvn spring-boot:run

cd CMS2.0/cms-portal-frontend

Full-Stack Docker (Backend + Frontend in Containers too)

If you want everything containerized (no hot-reload):

# Build backend image:
cd CMS2.0/cms-backend
docker build -t cms-backend:local .

# Build frontend image:
cd CMS2.0/cms-portal-frontend
docker build --build-arg NG_CONFIG=production -t cms-portal-frontend:local .

# Run backend (connects to Oracle in docker-compose network):
docker run -d --name cms-backend-app \
  --network docker_cms-network \
  -p 8082:8082 \
  -e DB_URL="jdbc:oracle:thin:@cms-oracle:1521/XEPDB1" \
  -e DB_USERNAME=cms_app \
  -e DB_PASSWORD=cms_app_password \
  -e SPRING_JPA_DATABASE_PLATFORM=org.hibernate.dialect.OracleDialect \
  -e SPRING_DATASOURCE_DRIVER_CLASS_NAME=oracle.jdbc.OracleDriver \
  -e KEYCLOAK_URL=http://cms-keycloak:8180 \
  -e KAFKA_BOOTSTRAP=cms-kafka:9092 \
  -e OPENSEARCH_URL=http://cms-opensearch:9200 \
  cms-backend:local

# Run frontend:
docker run -d --name cms-portal-app \
  --network docker_cms-network \
  -p 4300:80 \
  -e API_BACKEND_HOST=cms-backend-app \
  -e API_BACKEND_PORT=8082 \
  cms-portal-frontend:local

Stop & Cleanup

# Stop all containers (preserve data):
docker-compose down

# Stop and delete all data volumes:
docker-compose down -v

5. Option C — OpenShift Deployment

Setup Time: ~45 minutes

Full production-grade deployment on OpenShift 4.x with 16 microservices, auto-scaling, health probes, and TLS routes.

Step 1: Prerequisites Checklist

  • OpenShift cluster 4.x accessible
  • oc login completed with admin or project-admin privileges
  • Nexus repository available for Maven artifacts
  • Oracle Database (19c/21c) — external or in-cluster instance provisioned with PDB
  • External Kafka cluster or Strimzi operator installed
  • Keycloak with cms realm already configured

Step 2: Build Order (Maven Artifacts)

# Step 2a: Publish parent POM to Nexus
cd CMS2.0
mvn deploy -N -s deployment/maven-settings.xml

# Step 2b: Publish cms-common library to Nexus
cd cms-common
mvn deploy -s ../deployment/maven-settings.xml

# Step 2c: All microservices can now build in parallel

Step 3: Build & Push Container Images

# Login to OpenShift registry:
oc registry login

# Build all services (run from CMS2.0 root):
SERVICES="cms-backend cms-api-gateway cms-assignment-service cms-audit-service \
  cms-eligibility-service cms-ingestion-service cms-notification-service \
  cms-outbox-publisher cms-rules-service cms-search-service \
  cms-sla-monitor-service cms-storage-service cms-workflow-service \
  cms-paddle-ocr cms-portal-frontend cms-frontend"

for svc in $SERVICES; do
  echo "Building $svc..."
  ./deployment/build-and-push.sh $svc latest
done
For the frontend, the Dockerfile uses --build-arg NG_CONFIG=openshift by default, which sets apiBaseUrl: '' (relative URLs proxied by Nginx).

Step 4: Configure Secrets

Edit these files in deployment/openshift/ before applying:

# 1. Database credentials:
vi deployment/openshift/secret-db.yaml
# Change: DB_USER, DB_PASSWORD, SMTP_USER, SMTP_PASSWORD

# 2. Keycloak credentials:
vi deployment/openshift/secret-keycloak.yaml
# Change: KEYCLOAK_CLIENT_ID, KEYCLOAK_CLIENT_SECRET, KEYCLOAK_ADMIN_USER, KEYCLOAK_ADMIN_PASSWORD

# 3. SMS Gateway:
vi deployment/openshift/secret-sms.yaml
# Change: SMS_API_KEY, SMS_SENDER_ID, SMS_TEMPLATE_ID

# 4. OpenSearch:
vi deployment/openshift/secret-opensearch.yaml
# Change: OPENSEARCH_USER, OPENSEARCH_PASSWORD

# 5. Kafka (if SASL enabled):
vi deployment/openshift/secret-kafka.yaml
# Change: KAFKA_SASL_USERNAME, KAFKA_SASL_PASSWORD

Step 5: Update ConfigMap

# Edit configmap with your cluster-specific values:
vi deployment/openshift/configmap-common.yaml

# Key values to update:
#   DB_HOST: your-oracle-host.svc.cluster.local
#   KAFKA_BOOTSTRAP_SERVERS: your-kafka.svc.cluster.local:9092
#   KEYCLOAK_ISSUER_URI: https://your-keycloak/realms/cms
#   OPENSEARCH_HOST: your-opensearch.svc.cluster.local

Step 6: Update Route Hostnames

# Edit portal frontend route:
vi deployment/openshift/deployment-portal-frontend.yaml

# Change spec.host to your actual domain:
#   spec:
#     host: cms-portal.apps.your-cluster.example.com

Step 7: Deploy Everything

cd CMS2.0/deployment
chmod +x deploy-all.sh

6. Keycloak Realm Setup

Required for RBIO officer login, staff SSO, and CRPC workflows.

Create the CMS Realm

  1. Open http://localhost:8180 → Login with admin/admin
  2. Click dropdown (top-left) → Create Realm → Name: cms → Create
  3. Go to Clients → Create Client:
    • Client ID: cms-portal
    • Client authentication: OFF (public client)
    • Valid redirect URIs: http://localhost:4300/*
    • Web origins: http://localhost:4300
  4. Go to Realm Roles → Create these roles:
    • RBIO_OFFICER
    • RBIO_SUPERVISOR
    • CRPC_DEO
    • CRPC_REVIEWER
    • CRPC_HEAD
    • CEPC_OFFICER
    • ADMIN
  5. Go to Users → Create test users:
    UsernamePasswordRole
    rbio.officer1Test@123RBIO_OFFICER
    rbio.supervisor1Test@123RBIO_SUPERVISOR
    crpc.deo1Test@123CRPC_DEO
    crpc.reviewer1Test@123CRPC_REVIEWER
    cepc.officer1Test@123CEPC_OFFICER
  6. For each user: Credentials tab → Set password → Turn OFF "Temporary"

8. Troubleshooting

Port Already in Use

# Find process using port 8082:
netstat -ano | findstr :8082

# Kill it:
taskkill /PID <PID> /F

# Or use PowerShell:
Get-Process -Id (Get-NetTCPConnection -LocalPort 8082).OwningProcess | Stop-Process -Force

Kafka Connection Timeout (Backend startup slow)

If Kafka isn't running, the backend waits 60s per Kafka call. Fix:

# Set fast timeout:
set KAFKA_MAX_BLOCK_MS=3000
set KAFKA_DELIVERY_TIMEOUT=5000
set KAFKA_REQUEST_TIMEOUT=3000

Oracle Connection Refused / ORA-12541

# Check Oracle listener is running:
lsnrctl status

# If not running, start it:
lsnrctl start

# Check Oracle service is running (Windows):
net start | findstr /i oracle
# If not: net start OracleServiceXE

# Verify connectivity:
sqlplus cms_app/cms_app_password@localhost:1521/XEPDB1

# If user doesn't exist, create it:
sqlplus system/oracle123@localhost:1521/XEPDB1
CREATE USER cms_app IDENTIFIED BY cms_app_password DEFAULT TABLESPACE USERS QUOTA UNLIMITED ON USERS;
GRANT CONNECT, RESOURCE, CREATE VIEW, CREATE SEQUENCE, CREATE TRIGGER TO cms_app;
EXIT;

Oracle ORA-01017: Invalid username/password

# Reset password:
sqlplus system/oracle123@localhost:1521/XEPDB1
ALTER USER cms_app IDENTIFIED BY cms_app_password ACCOUNT UNLOCK;
EXIT;

Keycloak "Invalid redirect_uri"

Ensure the Keycloak client cms-portal has:

  • Valid Redirect URIs: http://localhost:4300/*
  • Web Origins: http://localhost:4300 (or * for dev)

Angular Build Errors

# Clear cache and reinstall:
cd CMS2.0/cms-portal-frontend
rm -rf node_modules .angular
npm install
npm start

Docker Compose — Oracle Takes Too Long

Oracle XE first-time initialization can take 3-5 minutes. Wait for health check to pass:

# Watch Oracle logs:
docker logs -f cms-oracle

# When you see "DATABASE IS READY TO USE!" it's done

OpenShift Pod CrashLoopBackOff

# Check logs:
oc logs deployment/cms-backend -n rbi-cms --tail=50

# Common causes:
# 1. Secret not found → ensure secrets applied before deployments
# 2. DB unreachable → check ConfigMap DB_HOST value
# 3. OOM → increase memory limits in deployment YAML
