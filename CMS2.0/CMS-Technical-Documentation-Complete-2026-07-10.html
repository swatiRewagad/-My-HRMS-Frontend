2.3 Key Services

  • OtpService — OTP generation, expiry, rate-limiting (5/hour/mobile)
  • CaptchaService — Server-generated CAPTCHA images
  • CitizenSessionService — Stateless JWT session management
  • CooloffService — Progressive lockout (30s → 60s → 120s → 300s → 600s)
  • ComplaintService — Core complaint CRUD, filing, status transitions
  • GeoLocationService — MaxMind GeoIP integration for auto-location
  • AppealEligibilityService — Validates appeal conditions (30-day window, closed status)

2.4 Security Features

  • OTP with progressive cooloff (exponential backoff)
  • Server-side CAPTCHA (6 char, 5-min expiry)
  • Anti-automation filter (min 3s form submission time, velocity detection)
  • Rate limiting: 10 complaints/minute, 5 OTP resends/hour
  • PII encryption at rest (AES-256) via PiiDecryptionFilter

9. MRE (Maintainability Rules Engine) Module

Rules Engine Automated assessment of complaint maintainability based on RB-IOS 2021/2026 scheme conditions.

9.1 MRE Grounds (Questions Assessed)

GroundCheckWindow
Q13 - Prior RE ComplaintDid complainant first approach the RE?—
Q16 - Response WindowDid RE fail to respond within window?30 days (banks), 60 days (card networks)
Q17 - Dissatisfied with ResponseDid RE respond but complainant is dissatisfied?—
Limitation PeriodIs complaint within 3-year limitation?3 years from cause of action
Duplicate CheckIs same grievance pending elsewhere?—
JurisdictionDoes complaint fall under RBI jurisdiction?—

9.2 Service Classes

  • MaintainabilityRulesEngine — Core engine: evaluates all grounds, produces MreVerdict
  • MreEntityCoverageService — Validates entity type is covered under applicable scheme
  • MreProperties — Configurable windows: re-window-days=30, limitation-period-years=3, window-basis=BUSINESS

9.3 Verdict Output

{
  "signal": "GREEN" | "RED" | "AMBER",
  "grounds": [
    {"ground": "Q13_PRIOR_COMPLAINT", "verdict": "PASS", "reason": "..."},
    {"ground": "Q16_RESPONSE_WINDOW", "verdict": "FAIL", "reason": "..."}
  ],
  "determination": "MAINTAINABLE" | "NON_MAINTAINABLE" | "REQUIRES_MANUAL_REVIEW"
}

9.4 Configuration (application.yml)

cms:
  mre:
    version: 1
    re-window-days: 30
    npci-window-days: 30
    card-network-window-days: 60
    filing-deadline-days: 90
    limitation-period-years: 3
    window-basis: BUSINESS

20.2 Step-by-Step Setup

Option A: Manual Setup (without Docker)

1
Clone the repository
git clone <repo-url> My-HRMS-Frontend
cd My-HRMS-Frontend/CMS2.0
2
Start MySQL
# Create database and user
mysql -u root -p
CREATE DATABASE cms_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'cms_user'@'localhost' IDENTIFIED BY 'cms_password';
GRANT ALL PRIVILEGES ON cms_db.* TO 'cms_user'@'localhost';
FLUSH PRIVILEGES;
3
Start Keycloak
# Windows (from c:\tools\keycloak-26.0.0)
bin\kc.bat start-dev --http-port=8180

# Linux/Mac
bin/kc.sh start-dev --http-port=8180

# Access: http://localhost:8180 (admin/admin)
# Create realm: "cms"
# Create client: "cms-app" (public, redirect: http://localhost:4300/*)
4
Start OpenSearch (optional)
# Windows (from c:\tools\opensearch-2.18.0)
bin\opensearch.bat

# Or disable in application.yml:
# cms.similar-cases.enabled: false
5
Start Backend
cd cms-backend
mvn spring-boot:run -Dspring-boot.run.arguments="--DB_PASSWORD=cms_password"

# Or with environment variables:
set DB_PASSWORD=cms_password
mvn spring-boot:run

# Backend starts at http://localhost:8082
# Hibernate auto-creates tables (ddl-auto=update)
# PincodeDataInitializer loads 797 pincodes from CSV
# TranslationDataInitializer loads 435 translations from CSV
6
Start Frontend (Portal)
cd cms-portal-frontend
npm install
ng serve --port 4300

# Portal available at http://localhost:4300
7
Verify
# Backend health:
curl http://localhost:8082/actuator/health

# Translations loaded:
curl http://localhost:8082/api/translations/en

# Pincode lookup:
curl http://localhost:8082/api/location/pincode/400001

Option B: Docker Compose (Full Stack)

1
Start all infrastructure
cd cms-infra
docker-compose up -d

# This starts:
#   Oracle DB       → localhost:1521
#   Kafka (KRaft)   → localhost:9092
#   Keycloak        → localhost:8180
#   OpenSearch      → localhost:9200
#   Kafka UI        → localhost:8090
#   Prometheus      → localhost:9090
#   Grafana         → localhost:3000
2
Run the seed script (for Oracle)
sqlplus cms_app/cms_app_password@//localhost:1521/XEPDB1 @CMS-Database-Init-2026-07-09.sql
3
Start backend with Oracle profile
cd cms-backend
mvn spring-boot:run -Dspring-boot.run.profiles=oracle \
  -Dspring-boot.run.arguments="--DB_URL=jdbc:oracle:thin:@localhost:1521/XEPDB1 --DB_USERNAME=cms_app --DB_PASSWORD=cms_app_password"
