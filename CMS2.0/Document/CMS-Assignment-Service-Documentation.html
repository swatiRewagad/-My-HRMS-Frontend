1. Overview

The CMS Assignment Service is a Spring Boot microservice responsible for automatically assigning incoming complaints to the appropriate team and officer based on configurable business rules. It uses Drools (KIE) as a rules engine to evaluate complaint attributes and determine the best assignment.

Key Responsibilities:
  • Listen for complaint.ingested Kafka events
  • Evaluate complaint category, priority, and amount against Drools rules
  • Assign complaint to the correct team (ATM_TEAM, DIGITAL_TEAM, etc.)
  • Auto-escalate HIGH/CRITICAL priority complaints
  • Publish complaint.assigned Kafka event for downstream services

Detailed Steps:

  1. Complaint Ingested: A new complaint is filed through any channel (web portal, email, physical letter). The ingestion service publishes a complaint.ingested event to Kafka.
  2. Listener Receives Event: ComplaintIngestedAssignmentListener consumes the event, extracts category and priority from the payload.
  3. AssignmentService Invoked: Creates an AssignmentFact with complaint attributes and fires Drools rules.
  4. Rules Evaluated: Drools rules match on category and priority, setting assignedTeam, assignedOfficer, and escalated flag.
  5. Result Published: A complaint.assigned event is published to Kafka with the assignment result, updating the complaint status to ASSIGNED.

4. Drools Rules Engine

The service uses KIE (Knowledge Is Everything) container from the Drools framework. Rules are loaded from src/main/resources/rules/assignment-rules.drl.

How it works:

// 1. Create a new KIE session
KieSession kieSession = kieContainer.newKieSession();

// 2. Insert the fact (complaint data)
kieSession.insert(fact);  // AssignmentFact with category, priority, amount

// 3. Fire all matching rules
kieSession.fireAllRules();

// 4. Read results from the mutated fact
// fact.getAssignedTeam() -> "ATM_TEAM"
// fact.isEscalated() -> true/false

// 5. Dispose session
kieSession.dispose();
Rule Salience (Priority): Rules with higher salience values fire first. Category rules have salience 100, escalation rules have 50, and the default fallback has 10. This ensures category-specific assignment happens before escalation check, and the default only fires if no category matched.

6. Round-Robin DEO Assignment

For email-ingested complaints, the system additionally assigns a Data Entry Operator (DEO) using a round-robin algorithm. This is handled by RoundRobinAssignmentService in the ingestion service.

Algorithm:

public String assignNextDeo() {
    // 1. Get all eligible DEOs (active, not on leave, below threshold)
    List<DeoUser> eligible = deoUserRepository.findEligibleDeos();

    // 2. Use atomic pointer for thread-safe round-robin
    int pointer = nextPointer.getAndUpdate(i -> (i + 1) % eligible.size());
    int index = pointer % eligible.size();

    // 3. Select DEO and increment their assigned count
    DeoUser selected = eligible.get(index);
    selected.setCurrentAssignedCount(selected.getCurrentAssignedCount() + 1);
    deoUserRepository.save(selected);

    return selected.getUserId();
}

DEO Eligibility Criteria:

  • Active: isActive = true
  • Not on Leave: isOnLeave = false
  • Below Threshold: currentAssignedCount < maxThreshold
Thread Safety: The round-robin pointer uses AtomicInteger to ensure thread-safe assignment even under concurrent Kafka message processing. However, the DEO count update is transactional at the database level.

10. Escalation Logic

Escalation is determined by the Drools rules engine and operates independently of team assignment:

ConditionEscalation LevelEffect
Priority = HIGH or CRITICALLevel 1Flagged for senior officer review
Amount > threshold (configurable)Level 1-2Reserved for future rules
SLA breached (30+ days)Level 2Handled by separate SLA service
Adding New Rules: To add a new assignment rule, edit src/main/resources/rules/assignment-rules.drl. No code changes or redeployment required if using KIE hot-reload. Set appropriate salience to control rule priority.

Example: Adding a new category rule

rule "Insurance Complaints - Assign to Insurance Team"
    salience 100
    when
        $fact : AssignmentFact(category == "INSURANCE")
    then
        $fact.setAssignedTeam("INSURANCE_TEAM");
        $fact.setAssignedOfficer(null);
        update($fact);
end
