Coding Standards

Backend (Java/Spring)

  • All entities use @Builder + @NoArgsConstructor + @AllArgsConstructor
  • Services are @RequiredArgsConstructor (constructor injection via Lombok)
  • Use @Transactional(readOnly = true) for read operations
  • Controllers return ResponseEntity<T>
  • Role-based access via @PreAuthorize("hasAnyRole('ROLE1', 'ROLE2')")
  • Async operations annotated with @Async
  • Scheduled tasks use @Scheduled(cron = "...")

Frontend (Angular/TypeScript)

  • All new components must be standalone: true
  • Use Angular Signals (signal(), computed()) for state management
  • Use inject() function instead of constructor injection
  • Lazy-load all route components
  • Services use providedIn: 'root'
  • Use @if / @for control flow (Angular 17+ syntax)
  • Style with component-scoped SCSS, no global styles

Git Conventions

  • Branch naming: feature/crpc-<feature-name>, fix/crpc-<bug-name>
  • Commit messages: feat(crpc): add notification bell component
  • PR title: Short (<70 chars), description explains WHY
