1. Overview & Architecture

CMS 2.0 has two separate Playwright projects serving different purposes:

1.3 Key Design Decisions

  • Page Object Model (POM) — All locators encapsulated in page classes
  • Two auth strategies — UI login via Keycloak or direct token injection
  • API helpers — Backend verification without going through UI
  • Test data fixtures — Pre-defined complaint payloads for deterministic testing
  • Graceful degradation — Tests skip if Keycloak is unavailable

5. Configuration Deep Dive

5.1 cms-e2e-tests/playwright.config.ts

import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  testDir: './tests',
  fullyParallel: true,            // Tests run in parallel by default
  forbidOnly: !!process.env.CI,   // Prevent .only() in CI
  retries: process.env.CI ? 2 : 0,// 2 retries in CI, 0 locally
  workers: process.env.CI ? 1 : undefined, // Sequential in CI

  reporter: [
    ['html', { open: 'never' }],  // HTML report (never auto-opens)
    ['list'],                      // Console output
  ],

  use: {
    baseURL: process.env.BASE_URL || 'http://localhost:4200',
    trace: 'on-first-retry',      // Capture trace on first retry
    screenshot: 'only-on-failure',// Screenshot on test failure
    video: 'retain-on-failure',   // Video recording retained on failure
  },

  projects: [
    { name: 'chromium', use: { ...devices['Desktop Chrome'] } },
    { name: 'firefox', use: { ...devices['Desktop Firefox'] } },
    { name: 'mobile-chrome', use: { ...devices['Pixel 5'] } },
  ],

  webServer: {
    command: 'cd ../cms-portal-frontend && ng serve',
    url: 'http://localhost:4200',
    reuseExistingServer: !process.env.CI, // Reuse if already running locally
    timeout: 120_000,                      // 2 min to start Angular
  },
});

6. Page Object Model (POM)

6.1 Pattern

Each page class encapsulates locators and actions for a specific page/view. Tests never use raw locators directly.

// pages/file-complaint.page.ts
import { type Locator, type Page, expect } from '@playwright/test';

export class FileComplaintPage {
  readonly page: Page;
  readonly nameInput: Locator;
  readonly submitButton: Locator;

  constructor(page: Page) {
    this.page = page;
    this.nameInput = page.getByLabel(/full name|name/i);
    this.submitButton = page.getByRole('button', { name: /submit|file complaint/i });
  }

  async goto() {
    await this.page.goto('/file-complaint');
  }

  async fillPersonalDetails(data: { name: string; email: string; phone: string }) {
    await this.nameInput.fill(data.name);
    // ... more fields
  }

  async submit() {
    await this.submitButton.click();
  }

  async expectSuccess() {
    await expect(
      this.page.getByText(/complaint.*registered|successfully.*filed/i)
    ).toBeVisible({ timeout: 10000 });
  }
}

6.3 Locator Strategy (Priority Order)

  1. page.getByRole() — Most resilient (button, link, heading)
  2. page.getByLabel() — For form inputs via associated labels
  3. page.getByText() — For assertions on visible text
  4. page.locator('[data-testid="..."]') — For complex components
  5. page.locator('.css-class') — Last resort, least stable
Convention: All locators use regex patterns (/pattern/i) for resilience against text changes and i18n translations.

7. Helpers & Fixtures

7.1 Authentication Helper (helpers/auth.ts)

Two authentication strategies depending on test needs:

Strategy 1: UI Login (Full Keycloak flow)

import { loginViaKeycloak, USERS } from '../helpers/auth';

test.beforeEach(async ({ page }) => {
  await loginViaKeycloak(page, USERS.rbioOfficer);
});
// Navigates to /, clicks login, fills Keycloak form, waits for redirect

Strategy 2: Token Injection (Faster, skips UI)

import { loginWithToken, USERS } from '../helpers/auth';

test.beforeEach(async ({ page }) => {
  await loginWithToken(page, USERS.rbioOfficer);
  await page.goto('/rbio/tasks');
});
// Gets token via Keycloak token endpoint, injects into localStorage

9. Writing New Tests — Developer Guide

9.1 Step-by-Step: Add a New Test File

Step 1: Create the Page Object (if new page)

// pages/my-new-page.page.ts
import { type Locator, type Page, expect } from '@playwright/test';

export class MyNewPage {
  readonly page: Page;
  readonly heading: Locator;
  readonly saveButton: Locator;

  constructor(page: Page) {
    this.page = page;
    this.heading = page.getByRole('heading', { name: /my feature/i });
    this.saveButton = page.getByRole('button', { name: /save/i });
  }

  async goto() {
    await this.page.goto('/my-feature');
  }

  async expectLoaded() {
    await expect(this.heading).toBeVisible();
  }
}

Step 2: Export from barrel file

// pages/index.ts — add:
export { MyNewPage } from './my-new-page.page';

Step 3: Write the spec file

// tests/my-feature.spec.ts
import { test, expect } from '@playwright/test';
import { MyNewPage } from '../pages';
import { loginViaKeycloak, USERS } from '../helpers/auth';

test.describe('My Feature', () => {
  let myPage: MyNewPage;

  test.beforeEach(async ({ page }) => {
    await loginViaKeycloak(page, USERS.admin);
    myPage = new MyNewPage(page);
    await myPage.goto();
  });

  test('should display the page correctly', async () => {
    await myPage.expectLoaded();
  });

  test('should save data', async ({ page }) => {
    // Fill form...
    await myPage.saveButton.click();
    await expect(page.getByText(/saved|success/i)).toBeVisible();
  });
});

Step 4: Add script shortcut (optional)

// package.json — add to scripts:
"test:my-feature": "npx playwright test tests/my-feature.spec.ts"

9.2 Adding API-Only Tests

import { test, expect } from '@playwright/test';

test.describe('API: My Endpoint', () => {

10. Patterns & Best Practices

10.1 Locator Best Practices

// GOOD — Role-based (resilient)
page.getByRole('button', { name: /submit/i })
page.getByLabel(/email/i)
page.getByText(/success/i)

// GOOD — data-testid (explicit contract)
page.locator('[data-testid="complaint-grid"]')

// AVOID — Brittle CSS selectors
page.locator('.btn.btn-primary.submit-btn')  // ❌ Breaks on style changes
page.locator('#app > div:nth-child(3) > button')  // ❌ DOM-structure dependent

10.2 Waiting Strategy

// GOOD — Wait for specific condition
await expect(page.getByText('Loaded')).toBeVisible();
await page.waitForURL(/.*dashboard.*/);
await page.waitForResponse('**/api/complaints');

// AVOID — Arbitrary timeouts
await page.waitForTimeout(5000);  // ❌ Flaky, wastes time

10.3 Test Isolation

// Each test gets a fresh browser context (cookies, localStorage cleared)
// For tests that modify data, use test.describe.serial():

test.describe.serial('Workflow Chain', () => {
  test('step 1: create', async () => { /* ... */ });
  test('step 2: process', async () => { /* ... */ });
  test('step 3: close', async () => { /* ... */ });
});

10.4 Handling Keycloak Availability

// Skip tests gracefully when Keycloak is down
test('should show dashboard', async ({ page }) => {
  const keycloakUp = await isKeycloakAvailable(page);
  test.skip(!keycloakUp, 'Keycloak is not available');
  // ... test logic
});

10.5 Network Mocking (for Edge Cases)

// Simulate service failure
await page.route('**/api/v1/copilot/maintainability/**', (route) =>
  route.fulfill({ status: 503, body: 'Service Unavailable' })
);

// Simulate slow response
await page.route('**/api/complaints', async (route) => {
  await new Promise((r) => setTimeout(r, 5000));
  await route.continue();
});

10.6 Assertion Patterns

// Visibility
await expect(locator).toBeVisible();
await expect(locator).toBeVisible({ timeout: 10000 }); // Custom timeout

// Text content
await expect(locator).toContainText(/MAINTAINABLE/);
await expect(locator).toHaveText('Exact Text');

// URL
await expect(page).toHaveURL(/.*dashboard.*/);

// Count
expect(await locator.count()).toBeGreaterThan(0);

// API response
expect(response.ok()).toBeTruthy();
expect(response.status()).toBe(404);

11. CI/CD Integration

11.1 CI Behavior (when CI=true)

  • forbidOnly: true — Fails build if any .only() left in code
  • retries: 2 — Retries failed tests twice before marking as failed
  • workers: 1 — Sequential execution (prevents resource contention)
  • webServer.reuseExistingServer: false — Starts fresh Angular server
  • trace: 'on-first-retry' — Captures trace on first retry for debugging
  • video: 'retain-on-failure' — Video saved for failed tests

12. Troubleshooting

12.1 Common Issues

ProblemCauseSolution
Tests fail with "Target closed" Angular dev server crashed or timeout Ensure ng serve is running. Increase webServer.timeout
Keycloak login timeout Keycloak not running or wrong URL Start Keycloak: bin\kc.bat start-dev --http-port=8180
"Complaint not found" in tests Sample data not seeded Run CMS-Database-Init-2026-07-09.sql to seed sample complaints
Locator not found (strict mode) Multiple elements match the locator Make locator more specific: add .first() or tighten regex
Tests pass locally, fail in CI Timing differences, headless rendering Add explicit waits: await expect(...).toBeVisible({timeout: 15000})
"Browser not installed" Playwright browsers not downloaded Run npx playwright install
CORS errors in API tests Backend CORS doesn't allow test origin Add test URL to cms.cors.allowed-origins in application.yml

12.4 Performance Tips

  • Use loginWithToken() instead of loginViaKeycloak() when testing non-login features (saves ~3s per test)
  • Group related tests with test.describe.serial() to share setup
  • API-only tests are fastest — use them for business logic verification
  • Avoid waitForTimeout() — use condition-based waits instead
  • Run specific files during development: npx playwright test tests/my-file.spec.ts
