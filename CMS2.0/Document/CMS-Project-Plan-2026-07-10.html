8. Dependencies & Risks

Critical Dependencies

#DependencyImpactMitigation
1 Kafka cluster availability Email syndication, event publishing, assignment service dev-local mode with DirectKafkaPublisher already implemented
2 Keycloak/Kavach UAM All authenticated flows (CRPC users, Admin) Mock auth in dev-local; test with real Kavach in staging
3 SMS Gateway (for OTP) Portal login, Tracking OTP verification Mock SMS service returning hardcoded OTP in dev-local
4 Digital Signature Certificate (DSC) UST95: Signed acknowledgement/closure letters Use test certificate; production DSC needed before UAT
5 OCR Service (Groq/PaddleOCR) CRPC email attachment processing Multi-provider failover already implemented
6 RBI Communication Templates UST258: "RBI must provide all template content before UAT" Use placeholder templates; swap with final content when provided

Risks

Risk 1 (High): jBPM 7.74 incompatibility with Hibernate 6 / Spring Boot 3.x. The PostgreSQL81Dialect issue may resurface in staging with real PostgreSQL. Mitigation: Upgrade to Kogito (jBPM successor) or use DevLocalWorkflowService pattern permanently with custom state machine.
Risk 2 (Medium): RBIOS 2026 auto-closure 16-question flow is highly complex with branching logic and sub-judice handling. May take longer than estimated. Mitigation: CR-J2 allocated full Sprint 7 for this; CR-A1 builds UI incrementally.
Risk 3 (Medium): Post-approval office routing with threshold management is a critical path for CRPC. Bugs here block downstream flows. Mitigation: Comprehensive unit tests + integration tests in Sprint 4-5; separate threshold reset logic.
Risk 4 (Low): Speech-to-text (UST86, UST304) relies on browser Web Speech API which has limited language support. Mitigation: Implement with graceful degradation; show "not supported" message for unsupported browsers/languages.
