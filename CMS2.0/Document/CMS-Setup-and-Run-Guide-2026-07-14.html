4. Option A: Without Docker Desktop (Bare Metal)

Use when: Kafka and Oracle are on a remote server, and you run services directly on your laptop via Maven/npm.

1 Set Environment Variables

Create a file cms-env.sh (or set in terminal):

# Oracle (remote server)
export DB_HOST=192.168.x.x
export DB_PORT=1521
export DB_SERVICE_NAME=CMSPDB
export DB_USERNAME=cms_user
export DB_PASSWORD=yourpassword
export DB_HIBERNATE_DIALECT=org.hibernate.dialect.OracleDialect

# Kafka (remote server)
export KAFKA_BOOTSTRAP_SERVERS=192.168.x.x:9092

# Keycloak (local)
export KEYCLOAK_URL=http://localhost:9090
export KEYCLOAK_REALM=rbi-cms
export KEYCLOAK_ISSUER_URI=http://localhost:9090/realms/rbi-cms
export KEYCLOAK_JWK_SET_URI=http://localhost:9090/realms/rbi-cms/protocol/openid-connect/certs

# OpenSearch (local)
export OPENSEARCH_HOST=localhost
export OPENSEARCH_PORT=9200

# CMS Backend specific
export CMS_ENCRYPTION_SECRET=my-local-dev-secret-key-min-16-chars
export SPRING_PROFILES_ACTIVE=default

2 Start Infrastructure (Local)

Keycloak (port 9090)

cd c:/tools/keycloak-26.0.0
bin/kc.bat start-dev --http-port=9090

Then create realm rbi-cms, clients cms-portal & cms-officer-portal, and users with appropriate roles.

OpenSearch

cd c:/tools/opensearch-2.18.0
bin/opensearch.bat

3 Quick Start (Backend Only — No Kafka/Oracle)

Fastest way to get running: The dev-local profile uses H2 in-memory DB, no Kafka required. 
cd CMS2.0/cms-backend
mvn spring-boot:run -Dspring-boot.run.profiles=dev-local

This gives you:

  • Backend API at http://localhost:8082
  • H2 Console at http://localhost:8082/h2-console
  • Pre-configured encryption secret
  • Auto-created schema with seed data

6.2 Container Lifecycle — Step by Step

Phase 1: Image Build (CI/CD Pipeline)

# Backend services — Multi-stage Docker build
FROM eclipse-temurin:21-jdk-alpine AS build
WORKDIR /app
COPY pom.xml .
COPY src ./src
RUN apk add --no-cache maven && mvn clean package -DskipTests -q

FROM eclipse-temurin:21-jre-alpine
COPY --from=build /app/target/*.jar app.jar
EXPOSE 8082
ENTRYPOINT ["java", "-XX:+UseG1GC", "-XX:MaxRAMPercentage=75.0", "-jar", "app.jar"]

# Frontend services — RHEL8 nginx
FROM rebit-quay-quay-image-registry.apps.hocpclz.rebit.local/buildpiper/rhel8_nginx_124:amd64-9.7
COPY nginx.conf /etc/nginx/nginx.conf
COPY src/assets/config-template.json /opt/app-root/src/config-template.json
COPY /dist/cms-portal-frontend/browser/ /opt/app-root/src/
CMD ["sh", "-c", "envsubst < config-template.json > config.json && nginx -g 'daemon off;'"]

Phase 2: Pod Initialization

# When OpenShift creates a Pod, it:
# 1. Mounts ConfigMap as environment variables
#    envFrom:
#      - configMapRef: cms-common-config
#      - configMapRef: cms-backend-config
#
# 2. Mounts Secrets as environment variables
#    env:
#      - name: DB_PASSWORD
#        valueFrom:
#          secretKeyRef:
#            name: cms-db-secret
#            key: DB_PASSWORD
#
# 3. For BACKENDS: Runs java -jar app.jar
#    Spring Boot reads ${DB_HOST}, ${KAFKA_BOOTSTRAP_SERVERS}, etc.
#    from application.yml which references env vars.
#
# 4. For FRONTENDS: Runs envsubst < config-template.json > config.json
#    Then starts nginx serving the Angular build + config.json

Phase 3: Health Checks & Traffic

# Backend readiness probe
readinessProbe:
  httpGet:
    path: /actuator/health
    port: 8082
  initialDelaySeconds: 60
  periodSeconds: 10

# Frontend readiness probe
readinessProbe:
  httpGet:
    path: /healthz
    port: 8080
  initialDelaySeconds: 10
  periodSeconds: 5

# Once READY → Pod receives traffic via Service → Route

8. Troubleshooting

CMS_ENCRYPTION_SECRET must be set

Cause: Running without dev-local profile and no env var set.

Fix: Use dev-local profile: mvn spring-boot:run -Dspring-boot.run.profiles=dev-local

Or set: export CMS_ENCRYPTION_SECRET=my-secret-key-min-16-chars

Port 4200 already in use

Cause: Previous Angular dev server still running.

Fix: Kill the process: netstat -ano | findstr :4200 then taskkill /PID <pid> /F

Or use a different port: ng serve --port 4202

Keycloak port conflict with API Gateway (both 8080)

Cause: Both services default to port 8080.

Fix: Start Keycloak on 9090: bin/kc.bat start-dev --http-port=9090

Frontend config.json 404 error

Cause: config.json is only generated in Docker/OpenShift via envsubst.

Fix: Locally, Angular uses environment.ts. Ignore the 404 — it's expected in dev mode.

Oracle connection refused in Docker

Cause: Oracle takes 60-120s to initialize on first run.

Fix: Wait for: docker-compose logs -f oracle-db | grep "DATABASE IS READY"

Kafka consumer not receiving events

Cause: Topics not auto-created or outbox-publisher not running.

Fix: Ensure cms-outbox-publisher is running. Check: oc logs deployment/cms-outbox-publisher

OpenShift push rejected (secret scanning)

Cause: API keys or secrets committed in code.

Fix: Use environment variables / ConfigMaps for all secrets. Never hardcode.
