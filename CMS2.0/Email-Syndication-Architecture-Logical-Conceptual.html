2 LANGUAGE DETECTION & TRANSLATION
Script Detection
Unicode range analysis:
Devanagari, Bengali, Tamil, Telugu, Kannada, Malayalam, Gujarati, Urdu (Arabic), Punjabi, Odia

→
Is Vernacular?

YES→
LLM Translation

Strategy Pattern: Each provider implements OcrProvider interface. Chain order is configurable via cms.ocr.chain in application.yml. First provider to return non-empty fields wins. Unavailable providers (missing API key) are auto-skipped.
COMPONENTS Service & Component Map
Controller

EmailSyndicationApiController

  • POST /ingest — JSON email ingestion
  • POST /ingest-with-attachment — Multipart
  • POST /drafts/physical-letter — Manual letter
  • Orchestrates the full pipeline
  • Calls all services in sequence
Service

LanguageTranslationService

  • Unicode script pattern detection (11 languages)
  • 15% threshold for vernacular classification
  • Translation via Groq or Gemini LLM
  • Preserves financial data (names, amounts, dates)
  • Returns confidence score
Service

RuleBasedExtractor

  • Admin-configurable extraction rules from DB
  • Two pattern types: REGEX, KEYWORD
  • Source scope: SUBJECT, BODY, or BOTH
  • Priority ordering (first match per field wins)
  • 16 target fields supported
  • Transforms: UPPERCASE, LOWERCASE, TRIM
Service

OcrExtractionService

  • Orchestrator for OCR provider chain
  • Configurable via cms.ocr.chain
  • Strategy pattern with fallback
  • Auto-skips unavailable providers
  • Returns first non-empty result
Provider

OCR Providers (6 implementations)

  • GroqOcrProvider — Llama-4 Vision
  • GeminiOcrProvider — Gemini 2.0
  • OpenAiOcrProvider — GPT-4V
  • AzureDocIntelligenceOcrProvider
  • HuggingFaceOcrProvider
  • PaddleOcrProvider — Self-hosted
Entity

EmailDraft (Draft Complaint)

  • draftId, messageId, threadId
  • Complainant: name, phone, email, address, state, district, pincode
  • Complaint: subject, body, summary, category
  • Entity: name, type, amount
  • Language: detected, translated, confidence
  • OCR: processed flag, confidence, extracted JSON
  • Workflow: status, assignedTo, processedBy, deoDecision
Entity

ExtractionRule

  • ruleCode, ruleName, description
  • patternType: REGEX | KEYWORD
  • pattern: regex string or keyword list
  • targetField: one of 16 allowed fields
  • sourceScope: SUBJECT | BODY | BOTH
  • extractGroup: regex capture group index
  • priority: lower = higher precedence
  • isActive: enable/disable without deletion
External

Infrastructure

  • MySQL: EmailDraft, ExtractionRule, EmailDraftAttachment
  • Filesystem: C:/cms-attachments/email-drafts/{draftId}/
  • Keycloak: DEO user pool (role: DEO)
  • Groq API: OCR + Translation
  • Gemini API: OCR + Translation fallback
  • Kafka: complaint.ingested event (post-approval)
