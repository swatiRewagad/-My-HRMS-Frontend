Status Lifecycle
Status Actor Description Next Step
PENDING System Email received, no DEO available Wait for assignment
ASSIGNED System → DEO Round-robin assigned to DEO DEO opens & processes
IN_PROGRESS DEO DEO has opened and is editing the draft DEO submits decision
DRAFT DEO Physical letter manually entered, saved as draft DEO completes entry
PENDING_REVIEW DEO → Reviewer DEO assessed as maintainable, awaiting reviewer Reviewer approves/rejects
APPROVED_ROUTED Reviewer → System Reviewer approved, complaint created & routed RBIO/CEPC handles complaint
REJECTED Reviewer Reviewer rejected the draft No further action
NON_MAINTAINABLE DEO Not within RBI Ombudsman jurisdiction Closed
DUPLICATE System Same sender+subject as existing open complaint Linked to parent
CONVERTED System Legacy status (direct conversion without review) Complete
Key Components & APIs

Backend APIs

  • POST /ingest — Ingest email (JSON)
  • POST /ingest-with-attachment — Ingest with file (multipart)
  • POST /drafts/physical-letter — Physical letter entry
  • GET /queue — List drafts (filter: status, assignedTo)
  • GET /drafts/:id — Draft detail
  • PUT /drafts/:id — Update fields, status, decision
  • POST /drafts/:id/convert — Direct convert (legacy)
  • POST /drafts/:id/reassign — Reassign to another DEO
  • GET /stats — Queue statistics
  • GET /deo — DEO pool management
  • GET/POST /ignore-list — Manage ignore patterns

Frontend Routes (CRPC Portal)

  • /crpc/login — Keycloak SSO login
  • /crpc/home — DEO dashboard & queue
  • /crpc/draft/:id — DEO assessment form
  • /crpc/physical-letter — Physical letter entry
  • /crpc/reviewer — Reviewer queue
  • /crpc/reviewer/draft/:id — Reviewer assessment
  • /crpc/ops-head — Operations head dashboard
  • /email-syndication/simulator — Email simulator (dev)
  • /email-syndication/queue — Email queue view
  • /email-syndication/ignore-list — Ignore list manager

Intelligence Layer

  • OCR: Extracts name, phone, address, amount from attachments (PDF/JPG/PNG)
  • Rule-Based Extraction: Admin-defined regex/keyword rules for field extraction
  • Language Detection: Auto-detects 10 Indian languages
  • Vernacular Translation: Translates non-English emails to English
  • Duplicate Detection: Same sender + subject → links to parent
  • Ignore List: Email/domain patterns to auto-reject spam

Assignment & Routing

  • DEO Assignment: Round-robin across active DEOs (max threshold per DEO)
  • Complaint Routing: Entity name → Department (RBIO vs CEPC)
  • Officer Assignment: Region-filtered round-robin for RBIO; role-based for CEPC
  • Target Offices: RBIO-MUM, RBIO-DEL, RBIO-CHE, RBIO-KOL, CEPC
  • Kafka Events: complaint.ingested published for downstream services
