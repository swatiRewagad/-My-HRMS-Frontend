1. What are Extraction Rules?

Extraction Rules are pattern-matching definitions that automatically pull out specific data (like pincode, phone number, bank name) from incoming complaint emails and map them to complaint form fields.

Instead of a DEO manually reading every email and typing in details, rules do it automatically — before the email even reaches the DEO's queue.

2. How It Works (Flow)

Email arrives
→
Rules scan text
→
Fields auto-filled
→
AI/OCR fills gaps
→
DEO reviews
  • Step 1: Email arrives in the system
  • Step 2: All active extraction rules scan the email subject & body
  • Step 3: Matched data is auto-filled into complaint fields
  • Step 4: AI/OCR fills any remaining empty fields (from attachments)
  • Step 5: DEO verifies and submits
Rules run BEFORE AI. Rule-extracted values take priority over AI-extracted values.

3. Pattern Types

KEYWORD LIST — Easiest to use

Just list words/phrases separated by commas. If any of them appear in the email, it's a match.

Pattern: SBI, State Bank of India, HDFC, HDFC Bank, ICICI

Email says: "I have a complaint against HDFC Bank regarding my savings account."
Result: entityName = "HDFC Bank" ✓

REGEX — For structured data (numbers, emails, dates)

A pattern language that matches text structures like "any 6 digits" or "text@text.com".

Don't worry! Use the templates in the portal — they auto-fill regex patterns for you.

Think of regex as a "search pattern with wildcards". Here are the building blocks:

5. Pattern Breakdown: Pincode Example

// The full pattern:
(?:pin\s*(?:code)?|PIN|pincode|postal|zip|address)[:\s-]*([1-8]\d{5})\b
PartMeaningWhat it matches
(?:Start a group (don't capture this part)
pin\s*(?:code)?"pin" + optional spaces + optional "code"pin, pin code, pincode
|PINOR uppercase "PIN"PIN
|pincodeOR "pincode"pincode
|postalOR "postal"postal code
|zipOR "zip"zip
|addressOR "address"address
)End of keyword group
[:\s-]*Optional colon, spaces, or dash after keyword": ", "- ", " "
(START capturing (this is what gets extracted)
[1-8]First digit must be 1-8 (valid Indian pincode)4, 1, 7
\d{5}Followed by exactly 5 more digits00001, 10023
)END capturing
\bWord boundary (number must end here)
Matches:
• "Pincode: 400001" ✓
• "pin code - 110001" ✓
• "address 560034" ✓
• "PIN: 700001" ✓

Does NOT match:
• "amount 500000" ✗ (no keyword before it)
• "transaction 123456" ✗ (no keyword before it)
• "pincode 900001" ✗ (starts with 9, invalid)

7. Understanding Priority

Priority controls which rule runs first. Higher number = checked first.

PriorityUse ForExample
200+Specific/accurate rules (with keywords)Pincode with "Pincode:" keyword
100Normal rules (default)Phone number extraction
50Fallback/generic rulesAny 6-digit number as pincode guess
First match per field wins. If a priority-200 rule already extracted "complainantPincode", the priority-50 rule is skipped for that field.

Example: Two Pincode Rules

PriorityRuleWhen it matches
200Pincode (with keyword)"Pincode: 400001" — definite match
50Pincode (fallback)Any 6-digit number starting 1-8 — best guess

If the email says "Pincode: 400001", the priority-200 rule catches it. If it just says "Mumbai 400001" without any keyword, the fallback (priority 50) picks it up.

9. Tips & Best Practices

For Non-Technical Users

  • Always use the template dropdown — it fills the regex for you
  • Prefer Keyword List type for bank names, categories, and entity types
  • Use Test Rules button — paste a sample email and verify before saving
  • Set higher priority (200) for rules with specific keywords, lower (50) for generic fallbacks

Common Mistakes to Avoid

  • Don't create overlapping rules for the same field without priority. Two rules for "complainantPincode" at priority 100 — only the first one in the list will ever fire.
  • Don't use very generic patterns at high priority. E.g., \d{6} at priority 200 will match amounts and transaction IDs too.
  • Keyword rules are case-insensitive. "hdfc" will match "HDFC" and "Hdfc".

Strategy: Specific → Generic

Use a layered approach for best results:

LayerPriorityExample
1. Keyword-confirmed200"Pincode: 400001" — very accurate
2. Standard patterns100Phone number (always 10 digits starting 6-9)
3. Fallback guesses50Any 6-digit number might be a pincode
Remember: Rules auto-fill fields but the DEO always reviews before converting to a complaint. Even if a rule makes a wrong guess, the DEO can correct it.
