Understand the BPMN Elements

Each visual element maps to behavior in the jBPM engine

Visual ShapeBPMN TypeWhat It DoesKey Property
Green circle Start Event Process begins here —
Yellow box (gear icon) Business Rule Task Calls Drools rules automatically ruleFlowGroup="assignment"
Blue box (person icon) User Task Waits for human action potentialOwner: #{assignedTeam}
Diamond Exclusive Gateway Routes to one path based on condition Condition on outgoing flows
Green box (gear icon) Service Task Calls Java code automatically implementation: java class
Clock on border Boundary Timer Event Fires after duration (non-interrupting) timeDuration: P30D
Clock (standalone) Intermediate Timer Pauses process for duration timeDuration: P7D

Common Edits You'll Make

The most frequent workflow changes and how to do them in Camunda Modeler

Change SLA Timer (30 days → 15 days)

1. Click the clock icon on Officer Review
2. In Properties Panel → Timer → Duration
3. Change P30D to P15D
4. Save

Change Auto-Close Timer

1. Click the standalone timer (after Notify)
2. Properties → Timer → Duration
3. Change P7D to P14D
4. Save

Add a New Step (e.g., "Manager Approval")

1. From left palette, drag User Task
2. Place between existing steps
3. Delete old flow, draw new connections
4. Set properties (name, owner)
5. Save

Add Parallel Processing

1. Add Parallel Gateway (+ icon)
2. Draw flows to multiple tasks
3. Add joining Parallel Gateway
4. All paths must complete before join

Change Gateway Condition

1. Click the arrow leaving a gateway
2. Properties → Condition
3. Edit expression (MVEL syntax)
4. Example: #{amount > 100000}

Add Notification at Any Step

1. Drag Service Task from palette
2. Set implementation = java
3. Set class = NotificationHandler
4. Connect into the flow

Critical Properties to Configure

When editing, these properties MUST be set correctly

PropertyLocationValueWhy It Matters
Process ID Process properties (click canvas background) com.rbi.cms.complaint-lifecycle Code references this ID to start the process
Is Executable Process properties true jBPM won't execute non-executable processes
Package Name Process properties (tns:packageName) com.rbi.cms.workflow Drools package for rule resolution
Rule Flow Group Business Rule Task properties assignment Matches ruleFlowGroup in .drl files
Potential Owner User Task → Assignments #{assignedTeam} MVEL expression → evaluates at runtime
Timer Duration Timer Event → Timer Definition P30D (ISO 8601) P = period, D = days, H = hours, M = minutes
Important: Process ID Must Not Change The Java code in WorkflowService.java uses PROCESS_ID = "com.rbi.cms.complaint-lifecycle" to start the workflow. If you change the Process ID in Camunda Modeler, you MUST update the Java constant too.

Creating a NEW Workflow (e.g., Loan Complaints)

Step-by-step guide for adding a completely new process

  • In Camunda Modeler: File → New → BPMN Diagram
  • Click canvas → Set Process ID: com.rbi.cms.loan-complaint-lifecycle
  • Set Is Executable = true
  • Set Package Name = com.rbi.cms.workflow
  • Design your workflow (start → tasks → end)
  • Save to: cms-workflow-service/src/main/resources/processes/loan-complaint-lifecycle.bpmn2
  • Update ComplaintIngestedListener.java to route to new process by category
  • Create matching rules in Admin UI (category: LOAN_ASSIGNMENT)
  • Restart workflow service
  • Test end-to-end with a loan complaint
# New file location: cms-workflow-service/ └── src/main/resources/ └── processes/ ├── complaint-lifecycle.bpmn2 # existing (general) └── loan-complaint-lifecycle.bpmn2 # new workflow
