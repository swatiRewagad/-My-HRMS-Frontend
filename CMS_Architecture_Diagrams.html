Service Communication Matrix

Synchronous (REST/HTTP)

  • Frontend → API Gateway → Services
  • Gateway validates JWT, routes to service
  • Ingestion → Keycloak (user lookup)
  • Workflow → Assignment (officer assign)

Asynchronous (Kafka Events)

  • Ingestion → complaint.created
  • Workflow → complaint.assigned
  • SLA Monitor → sla.breach
  • Outbox Publisher → All topics (reliable)
  • Notification consumes: notification.send
  • Audit consumes: audit.log
  • Search consumes: complaint.* (re-index)

Outbox Pattern

  • Services write to OUTBOX table in same TX
  • Outbox Publisher polls every 5 seconds
  • Publishes to Kafka with exactly-once semantics
  • Batch size: 100 | Max retries: 5
  • Ensures data consistency across services

Deployment Specifications

Container Platform

  • OpenShift Container Platform 4.x
  • Namespace: rbi-cms
  • Registry: registry.rbi.internal/cms/
  • Java 21 (Eclipse Temurin)
  • Node 22 (LTS) for Angular build
  • Nginx 1.27-alpine for static serving

Network Configuration

  • External: cms.rbi.org.in (TLS 1.3)
  • Internal: ClusterIP services
  • mTLS between all pods
  • NetworkPolicy: namespace isolation
  • F5 WAF at perimeter
  • IP whitelisting for admin APIs

Scaling Strategy

  • HPA: CPU > 70% triggers scale-out
  • Min 2 replicas (HA) for critical services
  • Singleton for SLA Monitor + Outbox (leader election)
  • Database: Connection pool per pod (20-50)
  • Kafka: Partition count = max consumers

Backup & DR

  • Oracle: RMAN daily full + hourly incremental
  • RPO: 1 hour | RTO: 4 hours
  • Kafka: MirrorMaker 2 to DR
  • File storage: Rsync every 6 hours
  • DR site: Active-passive (warm standby)
