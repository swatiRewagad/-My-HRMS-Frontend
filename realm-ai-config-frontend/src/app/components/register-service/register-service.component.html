Service Registry / Register New Service

Register New Service

Add a new service to the Kavach Service Registry. All fields marked with * are required.

1

Service Identity

Enter the service name and base URL — other details will be auto-populated from the URL.

The base URL is the primary identifier. Click Paste & Validate to auto-populate registry, version, and alternatives.
2

Service Details

Define the service naming, category, version, and description.

Unique identifier, auto-generated from name.
3

Endpoint Configuration

Configure the health check path. The base URL is defined in Section 1.

Relative path appended to the base URL for health monitoring.
4

Authentication

Specify how consumers should authenticate when calling this service.

5

Ownership & Status

Assign an owner and set the initial operational status of the service.

Optional tags for filtering and discovery.
Fields marked * are required before you can register.
